# <a name="how-to-define-value-equality-for-a-type-c-programming-guide"></a><span data-ttu-id="42f28-102">Vorgehensweise: Definieren von Wertgleichheit für einen Typ (C#-Programmierleitfaden)</span><span class="sxs-lookup"><span data-stu-id="42f28-102">How to define value equality for a type (C# Programming Guide)</span></span>

<span data-ttu-id="42f28-103">Wenn Sie eine Klasse oder Struktur definieren, entscheiden Sie, ob es sinnvoll ist, eine benutzerdefinierte Definition der Wertgleichheit (oder Äquivalenz) für den Typ zu erstellen.</span><span class="sxs-lookup"><span data-stu-id="42f28-103">When you define a class or struct, you decide whether it makes sense to create a custom definition of value equality (or equivalence) for the type.</span></span> <span data-ttu-id="42f28-104">In der Regel implementieren Sie Wertgleichheit, wenn Objekte des Typs zu einer Auflistung hinzugefügt werden sollen oder wenn ihr Hauptzweck im Speichern einer Reihe von Feldern oder Eigenschaften besteht.</span><span class="sxs-lookup"><span data-stu-id="42f28-104">Typically, you implement value equality when objects of the type are expected to be added to a collection of some sort, or when their primary purpose is to store a set of fields or properties.</span></span> <span data-ttu-id="42f28-105">Sie können die Definition der Wertgleichheit auf einem Vergleich aller Felder und Eigenschaften im Typ oder auf einer Teilmenge aufbauen.</span><span class="sxs-lookup"><span data-stu-id="42f28-105">You can base your definition of value equality on a comparison of all the fields and properties in the type, or you can base the definition on a subset.</span></span>

<span data-ttu-id="42f28-106">In beiden Fällen und in beiden Klassen und Strukturen sollte Ihre Implementierung den fünf Garantien der Äquivalenz folgen (für die folgenden Regeln wird angenommen, dass `x`, `y` und `z` nicht NULL sind):</span><span class="sxs-lookup"><span data-stu-id="42f28-106">In either case, and in both classes and structs, your implementation should follow the five guarantees of equivalence (For the following rules, assume that `x`, `y` and `z` are not null):</span></span>  
  
1. <span data-ttu-id="42f28-107">`true` gibt `x.Equals(x)` zurück.</span><span class="sxs-lookup"><span data-stu-id="42f28-107">`x.Equals(x)` returns `true`.</span></span> <span data-ttu-id="42f28-108">Dies wird als reflexive Eigenschaft bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="42f28-108">This is called the reflexive property.</span></span>  
  
2. <span data-ttu-id="42f28-109">`x.Equals(y)` gibt denselben Wert zurück wie `y.Equals(x)`.</span><span class="sxs-lookup"><span data-stu-id="42f28-109">`x.Equals(y)` returns the same value as `y.Equals(x)`.</span></span> <span data-ttu-id="42f28-110">Die wird als symmetrische Eigenschaft bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="42f28-110">This is called the symmetric property.</span></span>  
  
3. <span data-ttu-id="42f28-111">Wenn `true` von `(x.Equals(y) && y.Equals(z))` zurückgegeben wird, wird `true` von `x.Equals(z)` zurückgegeben.</span><span class="sxs-lookup"><span data-stu-id="42f28-111">if `(x.Equals(y) && y.Equals(z))` returns `true`, then `x.Equals(z)` returns `true`.</span></span> <span data-ttu-id="42f28-112">Dies wird als transitive Eigenschaft bezeichnet.</span><span class="sxs-lookup"><span data-stu-id="42f28-112">This is called the transitive property.</span></span>  
  
4. <span data-ttu-id="42f28-113">Aufeinander folgende Aufrufe von `x.Equals(y)` geben immer denselben Wert zurück, es sei denn, die Objekte, auf die x und y verweisen, werden geändert.</span><span class="sxs-lookup"><span data-stu-id="42f28-113">Successive invocations of `x.Equals(y)` return the same value as long as the objects referenced by x and y are not modified.</span></span>  
  
5. <span data-ttu-id="42f28-114">Ein nicht-NULL-Wert ist ungleich NULL.</span><span class="sxs-lookup"><span data-stu-id="42f28-114">Any non-null value is not equal to null.</span></span> <span data-ttu-id="42f28-115">Die CLR prüft jedoch bei allen Methodenaufrufen auf NULL und löst eine `NullReferenceException` aus, wenn der `this`-Verweis NULL ist.</span><span class="sxs-lookup"><span data-stu-id="42f28-115">However, the CLR checks for null on all method calls and throws a `NullReferenceException` if the `this` reference would be null.</span></span> <span data-ttu-id="42f28-116">Daher löst `x.Equals(y)` eine Ausnahme aus, wenn `x` NULL ist.</span><span class="sxs-lookup"><span data-stu-id="42f28-116">Therefore, `x.Equals(y)` throws an exception when `x` is null.</span></span> <span data-ttu-id="42f28-117">Dadurch wird Regel 1 oder 2 gebrochen, abhängig vom Argument für `Equals`.</span><span class="sxs-lookup"><span data-stu-id="42f28-117">That breaks rules 1 or 2, depending on the argument to `Equals`.</span></span>

 <span data-ttu-id="42f28-118">Jede Struktur, die Sie definieren, besitzt bereits eine Standardimplementierung der Wertgleichheit, die von der <xref:System.ValueType?displayProperty=nameWithType>-Außerkraftsetzung der <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType>-Methode geerbt wurde.</span><span class="sxs-lookup"><span data-stu-id="42f28-118">Any struct that you define already has a default implementation of value equality that it inherits from the <xref:System.ValueType?displayProperty=nameWithType> override of the <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="42f28-119">Diese Implementierung verwendet Reflektion, um alle Felder und Eigenschaften im Typ zu untersuchen.</span><span class="sxs-lookup"><span data-stu-id="42f28-119">This implementation uses reflection to examine all the fields and properties in the type.</span></span> <span data-ttu-id="42f28-120">Obwohl diese Implementierung richtige Ergebnisse produziert, ist sie relativ langsam im Vergleich zu einer benutzerdefinierten Implementierung, die Sie speziell für den Typ schreiben.</span><span class="sxs-lookup"><span data-stu-id="42f28-120">Although this implementation produces correct results, it is relatively slow compared to a custom implementation that you write specifically for the type.</span></span>  
  
 <span data-ttu-id="42f28-121">Die Implementierungsdetails für Wertgleichheit unterscheiden sich für Klassen und Strukturen.</span><span class="sxs-lookup"><span data-stu-id="42f28-121">The implementation details for value equality are different for classes and structs.</span></span> <span data-ttu-id="42f28-122">Sowohl Klassen als auch Strukturen erfordern dieselben grundlegenden Schritte zur Implementierung der Gleichheit:</span><span class="sxs-lookup"><span data-stu-id="42f28-122">However, both classes and structs require the same basic steps for implementing equality:</span></span>  
  
1. <span data-ttu-id="42f28-123">Überschreiben Sie die [virtuelle](../../language-reference/keywords/virtual.md) Methode <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="42f28-123">Override the [virtual](../../language-reference/keywords/virtual.md) <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="42f28-124">In den meisten Fällen sollte Ihre Implementierung von `bool Equals( object obj )` nur die typspezifische `Equals`-Methode aufrufen, die die Implementierung der Schnittstelle <xref:System.IEquatable%601?displayProperty=nameWithType> darstellt.</span><span class="sxs-lookup"><span data-stu-id="42f28-124">In most cases, your implementation of `bool Equals( object obj )` should just call into the type-specific `Equals` method that is the implementation of the <xref:System.IEquatable%601?displayProperty=nameWithType> interface.</span></span> <span data-ttu-id="42f28-125">(Siehe Schritt 2)</span><span class="sxs-lookup"><span data-stu-id="42f28-125">(See step 2.)</span></span>  
  
2. <span data-ttu-id="42f28-126">Implementieren Sie die Schnittstelle <xref:System.IEquatable%601?displayProperty=nameWithType>, indem Sie eine typspezifische `Equals`-Methode data-ttu-id="42f28-129">Lösen Sie keine Ausnahmen aus `Equals`aus.</span><span class="sxs-lookup"><span data-stu-id="42f28-129">Do not throw exceptions from `Equals`.</span></span> <span data-ttu-id="42f28-130">Nur für Klassen: Diese Methode sollte nur Felder prüfen, die in der Klasse deklariert sind.</span><span class="sxs-lookup"><span data-stu-id="42f28-130">For classes only: This Verweisgleichheit durchführt.)</span><span class="sxs-lookup"><span data-stu-id="42f28-132">(Do not do this if the type inherits directly from <xref:System.Object>, because the <xref:System.Object> 4. <span data-ttu-id="42f28-134">Überschreiben Sie <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType>, damit zwei Objekte, die Wertgleichheit besitzen, den gleichen Hashcode erzeugen.</span><span class="sxs-lookup"><span data-stu-id="42f28-134">Override <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType> so that two objects that have value equality produce the same hash code.</span></span>
