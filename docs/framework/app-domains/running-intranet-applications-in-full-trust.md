# <a name="running-intranet-applications-in-full-trust"></a><span data-ttu-id="f941f-102">Ausführen von Intranetanwendungen mit voller Vertrauenswürdigkeit</span><span class="sxs-lookup"><span data-stu-id="f941f-102">Running Intranet Applications in Full Trust</span></span>

<span data-ttu-id="f941f-103">Ab .NET Framework version 3.5 Service Pack 1 (SP1) können Anwendungen und deren Bibliothekassemblys als Assemblys mit voller Vertrauenswürdigkeit von einer Netzwerkfreigabe ausgeführt werden.</span><span class="sxs-lookup"><span data-stu-id="f941f-103">Starting with the .NET Framework version 3.5 Service Pack 1 (SP1), applications and their library assemblies can be run as full-trust assemblies from a network share.</span></span> <span data-ttu-id="f941f-104">Der Zonenbeweis <xref:System.Security.SecurityZone.MyComputer> wird automatisch zu Assemblys hinzugefügt, die aus einer Intranetfreigabe geladen wurden.</span><span class="sxs-lookup"><span data-stu-id="f941f-104"><xref:System.Security.SecurityZone.MyComputer> zone evidence is automatically added to assemblies that are loaded from a share on the intranet.</span></span> <span data-ttu-id="f941f-105">Durch diesen Beweis erhalten alle Assemblys den gleichen Satz an Berechtigungen (normalerweise volles Vertrauen) wie die Assemblys, die sich auf dem Computer befinden.</span><span class="sxs-lookup"><span data-stu-id="f941f-105">This evidence gives those assemblies the same grant set (which is typically full trust) as the assemblies that reside on the computer.</span></span> <span data-ttu-id="f941f-106">Diese Funktion gilt nicht für ClickOnce-Anwendungen oder für Anwendungen, die dafür entwickelt wurden, auf einem Host ausgeführt zu werden.</span><span class="sxs-lookup"><span data-stu-id="f941f-106">This functionality does not apply to ClickOnce applications or to applications that are designed to run on a host.</span></span>  
  
## <a name="rules-for-library-assemblies"></a><span data-ttu-id="f941f-107">Regeln für Bibliothekassemblys</span><span class="sxs-lookup"><span data-stu-id="f941f-107">Rules for Library Assemblies</span></span>  

<span data-ttu-id="f941f-108">Die folgenden Regeln gelten für Assemblys, die von einer ausführbaren Datei in einer Netzwerkfreigabe geladen wurden:</span><span class="sxs-lookup"><span data-stu-id="f941f-108">The following rules apply to assemblies that are loaded by an executable on a network share:</span></span>  
  
- <span data-ttu-id="f941f-109">Bibliotheksassemblys müssen sich im gleichen Ordner wie die ausführbaren Assemblys befinden.</span><span class="sxs-lookup"><span data-stu-id="f941f-109">Library assemblies must reside in the same folder as the executable assembly.</span></span> <span data-ttu-id="f941f-110">Assemblys, die sich in einem Unterordner befinden oder auf die mit einem anderen Pfad verwiesen wird, erhalten nicht das Berechtigungsset des vollen Vertrauens.</span><span class="sxs-lookup"><span data-stu-id="f941f-110">Assemblies that reside in a subfolder or are referenced on a different path are not given the full-trust grant set.</span></span>  
  
- <span data-ttu-id="f941f-111">Wenn die ausführbare Datei eine Assembly verzögert lädt, muss es den gleichen Pfad verwenden, der beim Starten der ausführbaren Datei verwendet wurde.</span><span class="sxs-lookup"><span data-stu-id="f941f-111">If the executable delay-loads an assembly, it must use the same path that was used to start the executable.</span></span> <span data-ttu-id="f941f-112">Wenn z.B. die Freigabe \\\\*network-computer*\\*share* einem Laufwerkbuchstaben zugeordnet wird und die ausführbare Datei von diesem Pfad ausgeführt wird, erhalten Assemblys, die von der ausführbaren Datei mithilfe des Netzwerkpfads geladen werden, kein volles Vertrauen.</span><span class="sxs-lookup"><span data-stu-id="f941f-112">For example, if the share \\\\*network-computer*\\*share* is mapped to a drive letter and the executable is run from that path, assemblies that are loaded by the executable by using the network path will not be granted full trust.</span></span> <span data-ttu-id="f941f-113">Um eine Assembly in der <xref:System.Security.SecurityZone.MyComputer>-Zone verzögert zu laden, muss die ausführbare Datei den Pfad des Laufwerkbuchstabens verwenden.</span><span class="sxs-lookup"><span data-stu-id="f941f-113">To delay-load an assembly in the <xref:System.Security.SecurityZone.MyComputer> zone, the executable must use the drive letter path.</span></span>

- <span data-ttu-id="f941f-122">Erstellen Sie auf 64-Bit-Computern einen Unterschlüssel unter dem Schlüssel „HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\\.NETFramework“ in der Registrierung des Systems.</span><span class="sxs-lookup"><span data-stu-id="f941f-122">On 64-bit computers, create a subkey under the HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\\.NETFramework key in the system registry.</span></span> <span data-ttu-id="f941f-123">Verwenden Sie den Schlüsselnamen „LegacyMyComputerZone“ mit einem DWORD-Wert von 1.</span><span class="sxs-lookup"><span data-stu-id="f941f-123">Use the key name LegacyMyComputerZone with a DWORD value of 1.</span></span> <span data-ttu-id="f941f-124">Erstellen Sie den gleichen Unterschlüssel unter dem Schlüssel „HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\\.NETFramework“.</span><span class="sxs-lookup"><span data-stu-id="f941f-124">Create the same subkey under the HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\\.NETFramework key.</span></span>
