# <a name="generating-a-wcf-client-from-service-metadata"></a><span data-ttu-id="70c53-102">Generieren eines WCF-Clients aus Dienstmetadaten</span><span class="sxs-lookup"><span data-stu-id="70c53-102">Generating a WCF Client from Service Metadata</span></span>

 <span data-ttu-id="70c53-104">Die Metadatendokumente können sich in einem permanenten Speicher befinden oder online abgerufen werden.</span><span class="sxs-lookup"><span data-stu-id="70c53-104">Metadata documents can be on a durable storage or be retrieved online.</span></span> <span data-ttu-id="70c53-105">Der Onlineabruf erfolgt gemäß dem WS-MetadataExchange-Protokoll oder Microsoft Discovery (DISCO)-Protokoll.</span><span class="sxs-lookup"><span data-stu-id="70c53-105">Online retrieval follows either the WS-MetadataExchange protocol or the Microsoft Discovery (DISCO) protocol.</span></span> <span data-ttu-id="70c53-106">Svcutil.exe gibt zum Abruf von Metadaten die folgenden Metadatenanforderungen gleichzeitig aus.</span><span class="sxs-lookup"><span data-stu-id="70c53-106">Svcutil.exe issues the following metadata requests simultaneously to retrieve metadata:</span></span>  
  
- <span data-ttu-id="70c53-107">WS-MetadataExchange (MEX)-Anforderung an die angegebene Adresse</span><span class="sxs-lookup"><span data-stu-id="70c53-107">WS-MetadataExchange (MEX) request to the supplied address.</span></span>  
  
- <span data-ttu-id="70c53-108">MEX-Anforderung an die angegebene Adresse mit angefügtem `/mex`</span><span class="sxs-lookup"><span data-stu-id="70c53-108">MEX request to the supplied address with `/mex` appended.</span></span>

 <span data-ttu-id="70c53-110">Svcutil.exe generiert den Client auf der Basis der vom Dienst empfangenen WSDL (Web Services Description Language)-Datei oder Richtliniendatei.</span><span class="sxs-lookup"><span data-stu-id="70c53-110">Svcutil.exe generates the client based on the Web Services Description Language (WSDL) or policy file received from the service.</span></span> <span data-ttu-id="70c53-111">Der Benutzer Prinzipal Name (User Principal Name, UPN) wird generiert, indem der Benutzername mit "" verkettet \@ und anschließend ein voll qualifizierter Domänen Name (FQDN) hinzugefügt wird.</span><span class="sxs-lookup"><span data-stu-id="70c53-111">The user principal name (UPN) is generated by concatenating the user name with "\@" and then adding a fully-qualified domain name (FQDN).</span></span> <span data-ttu-id="70c53-112">Für Benutzer, die sich bei Active Directory registriert haben, ist dieses Format jedoch nicht gültig, und der vom Tool generierte UPN verursacht einen Fehler bei der Kerberos-Authentifizierung mit der folgenden Fehlermeldung: Fehler **beim Anmeldeversuch.**</span><span class="sxs-lookup"><span data-stu-id="70c53-112">However, for users who registered on Active Directory, this format is not valid and the UPN that the tool generates causes a failure in the Kerberos authentication with the following error message: **The logon attempt failed.**</span></span> <span data-ttu-id="70c53-113">Um dieses Problem zu beheben, sollten Sie die von diesem Tool generierte Clientdatei manuell berichtigen.</span><span class="sxs-lookup"><span data-stu-id="70c53-113">To resolve this problem, manually fix the client file that the tool generated.</span></span>  
  
```console
svcutil.exe [/t:code]  <metadataDocumentPath>* | <url>* | <epr>  
```
