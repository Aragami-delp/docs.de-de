# <a name="conversion-from-type1-to-type2-cannot-occur-in-a-constant-expression-used-as-an-argument-to-an-attribute"></a><span data-ttu-id="5c391-102">Eine Konvertierung von "\<type2>" in "\<type1>" kann nicht in einem konstanten Ausdruck durchgeführt werden, der als Argument für ein Attribut verwendet wird</span><span class="sxs-lookup"><span data-stu-id="5c391-102">Conversion from '\<type1>' to '\<type2>' cannot occur in a constant expression used as an argument to an attribute</span></span>
<span data-ttu-id="5c391-103">Ein für ein Attribut Argument verwendeter Ausdruck wird zu einem Datentyp ausgewertet, der sich von dem des entsprechenden Attribut Parameters unterscheidet, und Visual Basic lässt die erforderliche Typkonvertierung für Attribut Argumente nicht zu.</span><span class="sxs-lookup"><span data-stu-id="5c391-103">An expression used for an attribute argument evaluates to a data type different from that of the corresponding attribute parameter, and Visual Basic does not allow the required type conversion for attribute arguments.</span></span>  
  
 <span data-ttu-id="5c391-104">Ein Attribut stellt Metadaten für das Element bereit, auf das es angewendet wird, und der Compiler muss zur Kompilierzeit in der Lage sein, alle Metadaten zu erstellen.</span><span class="sxs-lookup"><span data-stu-id="5c391-104">An attribute provides metadata for the element it is applied to, and the compiler must be able to construct all the metadata at compile time.</span></span> <span data-ttu-id="5c391-105">Daher muss jedes Attribut Werte verwenden, die zur Kompilierzeit konstant sind, und jedes Attributargument muss als ein zur Kompilierzeit konstanter Wert ausgewertet werden.</span><span class="sxs-lookup"><span data-stu-id="5c391-105">For this reason, every attribute must use values that are constant at compile time, and therefore every attribute argument must evaluate to a compile-time constant value.</span></span>  
  
 <span data-ttu-id="5c391-106">Bei bestimmten Typkonvertierungen können keine Werte erzeugt werden, die zur Kompilierzeit konstant sind.</span><span class="sxs-lookup"><span data-stu-id="5c391-106">Certain type conversions cannot produce values that are constant at compile time.</span></span> <span data-ttu-id="5c391-107">Ob beispielsweise `String` in `Double` oder `Date` konvertiert wird, hängt von der Gebietsschemaeinstellung zur Laufzeit ab.</span><span class="sxs-lookup"><span data-stu-id="5c391-107">For example, converting a `String` to a `Double` or a `Date` depends on the locale setting at run time.</span></span> <span data-ttu-id="5c391-108">Andere Konvertierungen (z. B. die Konvertierung eines Array eines abgeleiteten Typs in ein `Object`-Array) verursachen eine Vielzahl von Problemen, durch die der Compiler sie für Attributargumente nicht zulassen kann.</span><span class="sxs-lookup"><span data-stu-id="5c391-108">Other conversions, such as an array of a derived type to an array of `Object`, present a variety of problems that do not permit the compiler to allow them on attribute arguments.</span></span>
